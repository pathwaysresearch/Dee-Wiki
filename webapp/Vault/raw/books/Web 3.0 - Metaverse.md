# Introduction to Blockchain Technology

*   **Definition:**
    A distributed ledger technology that records transactions across multiple computers.

    A blockchain is a shared, distributed, append-only database that stores transactions or data across many computers — so that no single entity can alter it, and everyone can verify it.

*   **Key Characteristics:**
    *   Distributed
    *   Transparency
    *   Immutability
    *   Security

# Build a Blockchain – Blocks in a Chain

*   Divide yourselves into groups of 5 / 6 – Each one of you is a block
*   Each one of you has a card, you need to fill some details
*   Decide who is Block 1 and who is Block 5 / 6

[Page 18]
# Build a Blockchain – Recording a Transaction

*   Block Number: Fill your block number
*   Transaction: A transferred B Rs.10 (you can change names and amount)

Transaction Recorded in the Block

[Page 19]
# Build a Blockchain – Linking Blocks

*   **Current Hash** – Number of characters in your transaction (excluding spaces and punctuations) + Block number + Last two digits of previous hash
*   **Previous Hash** – Block 1: 0, Block 2 onwards – Copy current hash of previous block

Blocks Chained

[Page 20]
# Build a Blockchain – Tamper Proof

*   Block 2 in every group, increase the value of your transaction 10-fold (20 becomes 200)
*   Recalculate your current hash – cascade it downstream

Tamper proof - Immutable

[Page 21]
# Adding a Block – Proof of Work

*   Every group – Every block – Find the smallest 2-digit number when added to the block number + number of characters in the transaction ends in 0
    *   Smallest XY + Block Number + Total characters – ends in 0

Whoever gets the number first – can add a block. Adding a block gets you a reward – Proof of Work

[Page 22]
# Adding a Block – Consensus

*   The one who got the right to add a block – add a new transaction
*   All others – verify if the transaction is valid
    *   Block number, verify hash rule, previous hash, proof of work answer
    *   If good agree - Greater than 51% have to say yes

Consensus

| **New Need** | These are applications or platforms that address entirely new needs or create new markets. They often introduce novel technologies or paradigms. |
| **Timing the Market** | This column lists entities that are trying to capture a new market or redefine an existing one by being early or well-positioned. |
| **Native Apps (Most Upside)** | These are specific applications or services that are built for and thrive in a new paradigm (e.g., mobile-first, VR-first). They have the potential for significant growth if the new market takes off. |
| Google | (In the context of "New Need" and "Timing the Market") Likely refers to Google's early ventures into search, Android, or other new digital services that created new markets. |
| Uber | A prime example of a company that created a new market (ride-sharing) by leveraging mobile technology and a new business model. |
| Meta – Horizon Worlds | Meta's flagship VR social platform, aiming to create a new metaverse experience. It's a native app for a new VR/AR need. |
| **Familiar Need** | These are applications or platforms that address existing, well-understood human needs but might be adapting to new technologies or market conditions. |
| **Transitional Apps (Tough Spot)** | These are applications that serve familiar needs but are trying to adapt to new technological shifts. They might struggle if they don't fully embrace the new paradigm or if native solutions emerge. |
| MS Office | A suite of productivity tools for a familiar need (document creation, spreadsheets) that has had to adapt from desktop to cloud and mobile. |
| Netflix | Addresses a familiar need (entertainment) but revolutionized its delivery from physical media to streaming. Now faces competition from native streaming services. |
| EA – Game Titles | Traditional video game publisher adapting to new platforms (mobile, cloud gaming) and business models (free-to-play, subscriptions). |
| **Driving the Market** | This column lists entities that are building the foundational infrastructure, platforms, or ecosystems that enable new markets or technological shifts. They are often platform providers. |
| **Native Systems (Most Risk / Reward)** | These are the core platforms or operating systems that enable the "New Need" applications. They require significant investment and carry high risk, but offer immense reward if successful. |
| iTunes | Apple's platform for digital music distribution, which created a new market for legal digital music and laid groundwork for app stores. |
| App Store | Apple's platform for mobile application distribution, which created the modern mobile app economy. |
| Meta Quest Store | Meta's platform for VR application distribution, aiming to be the primary marketplace for VR content. |
| **Transitional Systems (Likely Winners)** | These are established platforms or infrastructure providers that are well-positioned to benefit from new technological shifts, even if they aren't creating the "new need" themselves. They provide essential services that new applications and systems rely on. |
| Windows | Microsoft's operating system, which has adapted over decades and remains a dominant platform for many applications. |
| AWS | Amazon Web Services, a cloud computing platform that provides the backend infrastructure for countless modern applications and services. |
| Nvidia | A company primarily known for GPUs, which are critical for AI, gaming, and metaverse applications, making them a key enabler of new technologies. |
| Unreal Engine | A powerful 3D creation tool and game engine, essential for developing high-fidelity games, simulations, and metaverse experiences. |

# Ethical and Social Implications

## Privacy and Security Concerns:
* Data Privacy: Ensuring user data is protected and not misused.
* Security: Protecting virtual spaces from hacking and malicious activities.

## Accessibility and Digital Divide:
* Access: Ensuring that everyone has access to the necessary technology and internet connectivity to participate in the metaverse.
* Inclusivity: Creating inclusive virtual spaces that represent diverse user groups.

## Mental Health and Social Issues:
* Addiction: Addressing the potential for addictive behavior in immersive environments.
* Social Isolation: Balancing virtual interactions with real-world connections.

## Examples:
* Facebook's Privacy Issues: Concerns about how user data is collected and used in virtual spaces.
* Accessibility Initiatives: Projects to make VR headsets more affordable and accessible.
