# Web 3.0 Metaverse

**Type**: RAG stub — full content in vector index, not in wiki

## Abstract

This source is a lecture/presentation by Vishal K from a "Frontier Tech – Session 3" series, covering the evolution of the web from Web 1.0 through Web 3.0, blockchain technology, and the metaverse. It uses accessible analogies (e.g., the internet as a highway system) and hands-on exercises (building a blockchain with participants as blocks) to explain foundational concepts. The presentation also examines the competitive "Metaverse Wars" (notably Meta/Horizon Worlds), current blockchain use cases across industries, and provides a framework for evaluating opportunities in frontier technologies by distinguishing between new needs vs. familiar needs and timing the market.

## Key Claims

- The internet functions as a giant system of highways connecting devices, governed by DNS for routing and protocols (HTTP, TCP/IP) as traffic rules
- Web paradigms evolved from Web 1.0 (Read Only Web, 1990s–early 2000s) through Web 2.0 to Web 3.0
- Blockchain is tamper-proof by design: changing a transaction in one block invalidates the hash chain of all subsequent blocks, making tampering detectable
- Blockchain has current real-world applications including crypto exchanges, copyright protection, securities exchanges, global payments, supply chain auditing, secure digital identities, and transparent voting
- Frontier technology opportunities can be evaluated along dimensions of new vs. familiar needs and transitional vs. native apps, with companies like Google, Uber, and Meta's Horizon Worlds serving as case studies

## Topics Covered

Web 1.0 history and characteristics, Web 2.0, Web 3.0, internet infrastructure and protocols (DNS/HTTP/TCP-IP), blockchain mechanics (hashing/chaining/tamper-proofing), blockchain use cases and current opportunities, metaverse concept and Metaverse Wars, Meta and Horizon Worlds, evaluating frontier technology opportunities, transitional apps vs. native apps, new needs vs. familiar needs framework, cryptocurrency exchanges and global payments
